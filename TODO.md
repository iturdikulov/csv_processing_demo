## Функциональные требования

1. Скрипт принимает пути к одному или нескольким CSV-файлам через аргумент --files . (обработка всех файлов)
2. Скрипт принимает название выходного отчета через аргумент --report . (опционально)
3. Формируется отчет "performance", содержащий: 
    ```
    Позицию ( position ) | Среднюю эффективность ( performance - среднее арифметическое по всем записям)
    ```
4. Отчет должен быть отсортирован по средней эффективности (по возрастанию/убыванию, подразумевается убывание для "эффективности").
5. Отчет выводится в консоль в виде таблицы (использовать tabulate ).

## Нефункциональные требования

1. Зависимости: 
    - Для обработки аргументов: только `argparse`
    - Для чтения CSV: только `csv`
    - Для всего остального (кроме тестирования и вывода): только стандартная библиотека Python.
2. Архитектура: Код должен поддерживать легкое добавление новых типов отчетов с использованием того же формата входных данных (CSV).
3. Тестирование: Код должен быть покрыт тестами, написанными на pytest . Рекомендуется покрытие ~80% критического функционала.
4. Стиль: Соответствие общепринятым стандартам и стилю Python (PEP 8).
